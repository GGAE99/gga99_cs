## Q. Docker란 무엇이며, 개발 환경에서 어떻게 활용하는가?

??? note "정답 보기 ▼"
    - [x] Linux Namespace + cgroup 기반 컨테이너 런타임으로, 이미지를 통해 재현 가능한 실행 환경을 제공
    - [ ] 게스트 OS를 통째로 띄워 강한 격리를 제공하는 하드웨어 가상화 하이퍼바이저(VMware/KVM 등)
    - [ ] 다수 노드 위에서 컨테이너의 배치·자동 복구·스케일링을 자동화하는 컨테이너 오케스트레이션 플랫폼
    - [ ] 프로세스의 루트 디렉터리만 격리하는 전통적인 Unix chroot/jail 기법으로 네트워크 격리는 미지원

### 핵심 답안 (30초)
> _컨테이너 런타임. Namespace + cgroup 기반 격리. 이미지를 통한 재현 가능한 환경._

## Q. AWS IAM을 사용하는 이유는?

??? note "정답 보기 ▼"
    - [x] 최소 권한 원칙으로 사용자·서비스별 권한을 분리하고 감사 가능하게 만들기 위해
    - [ ] AWS 리소스를 격리된 가상 네트워크 안에 배치하고 보안 그룹과 라우팅으로 트래픽을 제어하기 위해 (VPC)
    - [ ] 암호화 키를 안전하게 보관하고 회전·감사하여 데이터 암호화의 키 관리를 중앙화하기 위해 (KMS)
    - [ ] AWS API 호출 이력을 기록해 사후 감사와 비정상 행위 탐지에 사용하기 위해 (CloudTrail)

### 핵심 답안 (30초)
> _최소 권한 원칙. 사용자·서비스 단위 권한 제어. 자격 증명 분리와 감사._
