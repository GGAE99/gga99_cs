## Q. reflow와 repaint의 차이를 설명하시오.

??? note "정답 보기 ▼"
    - [x] Reflow는 레이아웃(위치·크기) 재계산으로 비용이 크고, Repaint는 픽셀만 다시 그리는 상대적으로 저렴한 작업
    - [ ] Reflow와 Repaint는 동일한 작업이며 명칭만 다르다
    - [ ] Repaint가 레이아웃 재계산이고 Reflow가 색상 변경이다 (정반대)
    - [ ] 둘 다 GPU에서 완전히 동일한 비용으로 처리된다

### 핵심 답안 (30초)
> _Reflow: 레이아웃 재계산(비싸다). Repaint: 픽셀 다시 그리기(상대적으로 저렴)._

## Q. CORS(Cross-Origin Resource Sharing)란 무엇이며, 왜 발생하는가?

??? note "정답 보기 ▼"
    - [x] 브라우저의 SOP(같은 출처 정책) 때문 — 다른 출처 자원 접근 시 서버가 허용 헤더를 응답해야 한다
    - [ ] 서버가 직접 차단하는 보안 정책으로, 브라우저는 관여하지 않는다
    - [ ] HTTPS 인증서가 다를 때만 발생하는 TLS 단계 문제
    - [ ] CDN이 응답을 캐싱하지 못해서 발생하는 캐시 정책 충돌

### 핵심 답안 (30초)
> _SOP(같은 출처 정책) 때문. 다른 출처 자원 접근 시 서버가 허용 헤더를 응답해야 함._

## Q. localStorage와 sessionStorage의 차이를 설명하시오.

??? note "정답 보기 ▼"
    - [x] localStorage는 영구 보관, sessionStorage는 탭 세션 종료 시 삭제 — 둘 다 도메인 단위, 5~10MB
    - [ ] sessionStorage가 영구 보관이고 localStorage가 세션 동안만 유지된다 (정반대)
    - [ ] 둘 다 매 요청마다 서버로 자동 전송된다 (쿠키와 혼동)
    - [ ] localStorage는 1MB, sessionStorage는 100MB로 용량 차이가 핵심이다

### 핵심 답안 (30초)
> _localStorage: 영구. sessionStorage: 탭 세션 종료 시 삭제. 둘 다 도메인 단위, 5~10MB._

## Q. 브라우저 메모리 캐시와 디스크 캐시의 차이는?

??? note "정답 보기 ▼"

    - [ ] 둘 다 동일한 저장소이며 명칭만 다르다
    - [ ] 디스크 캐시가 메모리 캐시보다 항상 빠르다
    - [ ] 메모리 캐시는 영구이고 디스크 캐시는 세션 단위이다 (정반대)

### 핵심 답안 (30초)
