## Q. 트랜잭션에 대해 간략하게 설명하시오.

??? note "정답 보기 ▼"
    - [x] 하나의 논리적 작업 단위로, ACID(원자성·일관성·격리성·지속성)를 보장하는 DB의 핵심 단위
    - [ ] 동시에 실행되는 작업 간 가시성을 제어하는 설정으로 Read Uncommitted ~ Serializable 4단계가 있는 격리 수준
    - [ ] 검색 속도를 높이기 위해 컬럼에 만드는 별도 자료구조로, 쓰기 비용 증가의 트레이드오프가 있는 인덱스
    - [ ] 데이터를 여러 노드에 동기/비동기로 복제해 가용성과 읽기 성능을 향상시키는 복제(replication) 기법

## Q. SQL과 NoSQL의 차이점을 설명하시오.

??? note "정답 보기 ▼"
    - [x] SQL은 정형 스키마·관계·강한 일관성, NoSQL은 유연한 스키마·수평 확장·다양한 데이터 모델을 특징으로 함
    - [ ] OLTP는 짧은 트랜잭션 위주의 운영 워크로드, OLAP는 대용량 집계와 분석에 최적화된 워크로드라는 용도 차이
    - [ ] 정규화는 중복을 제거해 일관성을 높이고, 비정규화는 조인을 줄여 읽기 성능을 끌어올리는 설계 트레이드오프
    - [ ] 행 지향 DB는 한 레코드의 모든 필드를 함께 저장하고, 열 지향 DB는 같은 컬럼끼리 묶어 분석에 유리

### 핵심 답안 (30초)
> _SQL: 정형 스키마·관계·강한 일관성. NoSQL: 유연한 스키마·수평 확장·다양한 모델(Doc/KV/Column/Graph)._

## Q. RDB와 NoSQL의 차이를 설명하시오.

??? note "정답 보기 ▼"
    - [x] 스키마 유연성, JOIN 가능 여부, 확장 방식(수직/수평), 일관성 모델에서 본질적 차이가 있음
    - [ ] ACID는 강한 일관성을 보장하는 트랜잭션 속성, BASE는 가용성을 우선해 결과적 일관성을 채택하는 접근의 차이
    - [ ] 1NF부터 BCNF까지 함수 종속성을 제거해 가는 정규화 단계의 깊이에 따른 설계 차이
    - [ ] OLTP 워크로드와 OLAP 워크로드 중 어느 쪽을 우선해서 인덱스 전략을 짜는지의 차이

### 핵심 답안 (30초)
> _스키마 유연성, JOIN 가능 여부, 확장 방식(수직/수평), 일관성 모델._
