## Q. SQL과 NoSQL의 차이점을 설명하시오.

??? note "정답 보기 ▼"
    - [x] SQL은 정형 스키마·관계·강한 일관성, NoSQL은 유연한 스키마·수평 확장·다양한 데이터 모델
    - [ ] SQL은 오픈소스이고 NoSQL은 상용 라이선스만 존재한다
    - [ ] SQL은 트랜잭션을 지원하지 않고 NoSQL만 트랜잭션을 지원한다
    - [ ] SQL은 쿼리 언어가 표준화돼 있지 않고 NoSQL이 표준화돼 있다

### 핵심 답안 (30초)
> _SQL: 정형 스키마·관계·강한 일관성. NoSQL: 유연한 스키마·수평 확장·다양한 모델(Doc/KV/Column/Graph)._

## Q. RDB와 NoSQL의 차이를 설명하시오.

??? note "정답 보기 ▼"
    - [x] 스키마 유연성, JOIN 가능 여부, 확장 방식(수직/수평), 일관성 모델에서 차이
    - [ ] 데이터를 저장하는 물리 디스크 포맷이 다르고 쿼리 언어는 동일하다
    - [ ] RDB는 인메모리 전용이고 NoSQL은 디스크 기반이다
    - [ ] RDB는 단일 노드만 가능하고 NoSQL은 무조건 분산 노드만 가능하다

### 핵심 답안 (30초)
> _스키마 유연성, JOIN 가능 여부, 확장 방식(수직/수평), 일관성 모델._
