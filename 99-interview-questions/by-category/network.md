## Q. URL과 URI에 대해 설명하시오.

??? note "정답 보기 ▼"
    - [x] URI = URL ∪ URN — URL은 위치, URN은 이름을 의미하며 모든 URL은 URI에 포함된다
    - [ ] URL이 상위 개념이고 URI가 그 하위 형태 (정반대)
    - [ ] URL과 URI는 동의어로 RFC가 두 명칭을 혼용한 결과 우연히 함께 존재한다
    - [ ] URN은 IP 주소만 표현 가능한 별개의 식별 체계로 URI와 무관하다

### 핵심 답안 (30초)
> _URI = URL ∪ URN. URL은 위치, URN은 이름. 모든 URL은 URI._

## Q. TCP와 UDP에 대해 설명하시오.

??? note "정답 보기 ▼"
    - [x] TCP는 연결지향·신뢰성·순서 보장, UDP는 비연결·빠름·손실 허용
    - [ ] TCP는 비연결이고 UDP가 연결지향이다 (정반대)
    - [ ] 둘 다 동일한 연결 절차를 거치며 차이는 포트 번호 범위뿐이다
    - [ ] TCP는 응용 계층, UDP는 전송 계층 프로토콜로 서로 다른 OSI 레이어다

### 핵심 답안 (30초)
> _TCP: 연결지향·신뢰성·순서 보장. UDP: 비연결·빠름·손실 허용._

## Q. TCP 3-way handshake에 대해 설명하시오.

??? note "정답 보기 ▼"
    - [x] SYN → SYN-ACK → ACK 순서로 시퀀스 번호를 동기화하며 연결을 수립
    - [ ] SYN → ACK 두 번으로 끝나는 2-way handshake가 표준이다
    - [ ] 데이터 페이로드 첫 패킷에 ACK 비트만 세팅하면 즉시 연결이 성립한다
    - [ ] 클라이언트가 RST를 보내 연결을 강제로 시작하는 절차다

### 핵심 답안 (30초)
> _SYN → SYN-ACK → ACK. 시퀀스 번호 동기화와 연결 수립._

## Q. HTTP의 주요 특성 2가지를 말하고 설명하시오.

??? note "정답 보기 ▼"
    - [x] Stateless(상태 없음) + Connectionless(원래 매 요청마다 연결 종료) — 확장성 ↑, 상태는 쿠키/세션/토큰으로 보완
    - [ ] Stateful + Connection-oriented — 서버가 항상 클라이언트 상태를 메모리에 보관
    - [ ] 양방향 풀 듀플렉스 + 바이너리 프레임 — 기본적으로 WebSocket과 동일
    - [ ] 멀티캐스트 + 신뢰성 없음 — UDP 위에서 직접 동작

### 핵심 답안 (30초)
> _Stateless + Connectionless(원래). 확장성 ↑, 그러나 상태 유지 필요시 쿠키/세션/토큰._

## Q. 웹소켓과 HTTP의 차이점을 설명하시오.

??? note "정답 보기 ▼"

    - [ ] WebSocket은 단방향 푸시 전용으로 클라이언트가 서버로 보낼 수 없다
    - [ ] 둘 다 매 메시지마다 새 TCP 연결을 맺는다
    - [ ] WebSocket은 UDP 기반이고 HTTP는 TCP 기반이라 다르다

### 핵심 답안 (30초)
> _HTTP는 요청-응답 단방향. WebSocket은 양방향 풀 듀플렉스. 초기 핸드셰이크는 HTTP Upgrade._

## Q. NAT(Network Address Translation)에 대해 설명하시오.

??? note "정답 보기 ▼"
    - [x] 사설 IP와 공인 IP 사이의 주소 변환 — IPv4 주소 부족 완화 + 내부 망 은닉 효과
    - [ ] DNS가 도메인을 IP로 바꾸는 작업과 정확히 동일한 메커니즘
    - [ ] MAC 주소를 IP 주소로 변환하는 ARP 프로토콜의 별칭
    - [ ] IPv6 환경에서 라우팅 효율을 높이기 위해 도입된 새로운 표준

### 핵심 답안 (30초)
> _사설 IP ↔ 공인 IP 변환. IPv4 주소 부족 완화 + 내부 망 은닉._

## Q. DNS는 무엇인가?

??? note "정답 보기 ▼"
    - [x] 도메인 ↔ IP 매핑을 제공하는 계층적 분산 시스템 — 재귀/반복 질의와 캐시 계층을 사용

### 핵심 답안 (30초)
> _도메인 ↔ IP 매핑. 계층적 분산 시스템. 재귀/반복 질의, 캐시 계층._

## Q. 리버스 프록시와 포워드 프록시의 차이점을 설명하시오.

??? note "정답 보기 ▼"

> _포워드: 클라이언트 측에서 대신 요청. 리버스: 서버 앞단에서 받아 분배._
