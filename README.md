# CS Knowledge Base

> 컴퓨터 과학 기초부터 실무까지, 계층적으로 정리한 학습 저장소

## 목적

- **체계적인 CS 지식 정리**: 흩어진 지식을 계층별로 정리
- **면접 대비**: 기술 면접 질문과 답변 정리
- **실무 연결**: 이론을 실제 프로젝트에 어떻게 적용하는지

## 학습 로드맵

학습은 **Layer 1 → 4** 순서를 권장하지만, 실무자의 경우 관심 영역부터 시작해도 좋습니다.

### Layer 1. 전공 CS 기초 (필수)
> "어떤 분야로 가든 필요한 공통 기반"

- [01-computer-science](./01-computer-science/) — 자료구조, 알고리즘, OS, 네트워크, DB, 컴퓨터 구조, SW 공학, 이론

### Layer 2. 언어 & 패러다임
> "도구의 동작 원리를 깊게 이해"

- [02-language](./02-language/) — JavaScript, TypeScript, Java, Kotlin, Python 등

### Layer 3. 실무 핵심 영역
> "프로덕션에서 마주하는 것들"

- [03-backend](./03-backend/) — 프레임워크, API 설계, 인증, 캐싱, 메시징
- [04-system-design](./04-system-design/) — 확장성, MSA, 분산 시스템, 케이스 스터디
- [05-devops-infra](./05-devops-infra/) — Docker, K8s, AWS, CI/CD, 모니터링
- [06-security](./06-security/) — 웹 보안, 암호학, 인증 심화

### Layer 4. 클라이언트 영역
> "사용자와 만나는 지점"

- [07-web](./07-web/) — 브라우저 렌더링, HTTP 심화, CORS, 성능
- [08-frontend](./08-frontend/) — React, Next.js, 상태관리, CSS, 빌드 도구
- [09-mobile](./09-mobile/) — iOS, Android, RN

### Layer 5. 선택 영역
- [10-ai-data](./10-ai-data/) — ML/LLM 기초, 빅데이터, 추천 시스템

## 작성 규칙

1. **하나의 주제는 하나의 파일**: 큰 주제는 폴더로 분리
2. **답을 외우지 말고 흐름을 정리**: "왜 이 기술이 등장했나?" → "어떻게 동작하나?" → "언제 쓰나?"
3. **실무 사례 포함**: 가능하면 직접 겪었거나 본 사례를 함께 기록
4. **링크는 풍부하게**: 출처와 추가 자료를 명시

## 업데이트 흐름

1. 학습 또는 면접 준비 중 새로운 주제 발견
2. 해당 Layer의 폴더에 마크다운 작성
3. 면접 질문 형식이 있으면 `99-interview-questions/`에도 추가
4. 섹션 README 인덱스 갱신
